# 核心能力设计

FormulaLab 是面向公式型知识的 Review-first 训练工具。产品优先帮助学习者完成当天该复习的公式、发现薄弱点，并把提示逐步沉淀成自己的回忆线索。

## 学习闭环

```text
公式内容库 -> 首次诊断 -> 今日复习 -> 结果反馈 -> 个人提示 -> 补弱与再调度
```

## 核心能力

1. **公式知识库**
   - 以公式为最小学习单元，记录表达式、变量、适用条件、误用边界、典型题型、例子和标签。
   - 支持按知识域、难度、标签和关键词浏览，并展示公式之间的前置、相关、易混、应用关系。

2. **诊断校准**
   - 从当前知识域抽取少量题目，让用户自评掌握程度。
   - 根据 `clear / partial / none` 标记薄弱公式，并初始化用户的公式训练状态。

3. **今日复习**
   - 根据每条公式的 `nextReviewAt` 生成今日队列。
   - 复习题型在回忆、识别、应用之间轮换，提交 `again / hard / good / easy` 后更新记忆强度、稳定度、难度估计和下次复习时间。

4. **补弱训练**
   - 将近期 `again`、`hard` 或稳定度较低的公式聚合成补弱队列。
   - 用户可以从总结页或复习页直接回到薄弱公式，优先修复短板。

5. **个人记忆钩子**
   - 每条公式允许用户保存一句自己的“下次提示”。
   - 复习请求提示时优先使用个人记忆钩子；没有钩子时退回公式的一句话用途。

6. **复习总结与指标**
   - 汇总最近训练轮次、评分分布、平均反应时间、薄弱公式、记忆钩子活动和长期进度。
   - 记录关键产品事件，用于判断弱项曝光、打开和训练转化。

7. **内容生产辅助**
   - 内部工作台为公式生成解释草稿、复习题和关系候选。
   - 草稿经人工审核后写入 approved seed 内容，用于持续扩充高质量公式库。

## 设计原则

- **复习优先**：登录后默认进入今日复习，公式库和工具页围绕训练任务展开。
- **按知识域隔离**：用户可切换学习域，诊断、复习、总结和路径都以当前知识域为上下文。
- **公式是内容中心，用户状态是训练中心**：公共公式内容保持稳定，个人掌握度、复习日志和记忆钩子按用户独立记录。
- **开发期直接演进**：当前阶段优先把目标行为做完整，数据结构变更直接调整 baseline。

## 信息架构

用户侧主导航收敛为三个学习意图：

1. **训练**：承载今日复习、弱项重练和诊断校准。
2. **公式库**：承载公式查找、学习路径和推导训练。
3. **进展**：承载训练结果、薄弱点、下次计划和个人提示整理。

诊断、补弱、路径、推导不作为顶层并列任务，而是在对应学习意图下按状态和场景出现。
